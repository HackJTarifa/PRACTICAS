    MODULO 2, PROGRAMACION SECUENCIAL CON FUNCIONES

      CAMBIOS MASTERMIND

    • Añadido updateHistorial() se encarga de actualiza el historial de tiradas y aciertos
    • Añadido printMessageEndGame() se encarga de imprimir por pantalla el nombre del juegador ganandor.
    • Añadido gameEnd() se encarga de comprovar si un jugador a ganado o ha llegado al limite de movimientos.
    • Añadido playerWin() se encarga de comprovar si la combiancion introducida es igual a la ganadora.
    • Añadido checkCombinacion() se encarga de comparar la combinacion ingresada con la ganadora, devuelve una array con colores blancos o negros o vacio dependiendo de si aciertas, existe el color o ninguna de las dos.
    • Añadido getCombinacion() se encarga de pedir al usuario una combinacion y testea si es valida.
    • Añadido getWinCombination() se encarga de crear una combinacion aleatoria con los colores validos, como combinacion ganadora.
    • Añadido playAgain() se encarga de preguntarle al usuario si quiere volver a jugar.

      CAMBIOS TIC TAC TOE

    • Añadido printTablero(), que se encarga de imprimir el tablero del tic tac toe.
    • Añadido playAgain(), que se encarga de preguntar al jugador si quiere volver a jugar.
    • Añadido printMessageEndGame(), que se encarga de imprimir por pantalla que jugador es el ganador.
    • Añadido gameEnd(), que se encarga de comprovar si el juego a terminado, hay ganador o no hay mas movimientos posibles.
    • Añadido playerWin(), que se encarga de comprovar si el jugador que hecho el movimiento ha ganado.
    • Añadido getFile(), que se encarga de pedir al jugador que elija una fila, y se encarga de comprovar que el valor ingresado es valido.
    • Añadido getColumn(), que se encarga de pedir al jugador que elija una columna, y se encarga de comprovar que el valor ingresado es valido.
    • Añadido isPositionValid(), que se encarga de comprovar que la posicion donde se quiere ingresar el valor no esta ya ocupada.
    • Añadido inicializeTablero(), que se encarga de crear una array con todas las posicion del tablero, y agrega un valor a cada posicición.

      CAMBIOS CONNECTA 4

    • Añadido updateTablero() se encarga de ingresar el TOKEN en una columna, se presupone que se ha comprovado antes que es una columna valida, comienza desde la posicion 0 hasta encontrar la primera posicion vacia valida.
    • Añadido isPosititionValid() se encarga de comprovar si una posicion del tablero el valida.
    • Añadido getColumn() se encarga de perdirle al jugador una colonumna para ingresar el TOKEN, esta tiene que ser valido 1...7.
    • Añadido printTablero() se encarga de imprimir el tablero con sus valores.
    • Añadido gameEnd() se encarga de comprovar si el jugador a ganado o no hay mas movimientos posibles.
    • Añadido playerWin() se encarga de comprovar si el jugador ha ganado.
    • Añadido printMessageEndGame() se encarga de imprimir por pantalla el mensaje de fin de juego, o jugador ganador.
    • Añadido playAgain() se encarga de pregunatar a los usuarios su quieren volver a jugar.
    • Añadido inicializeTablero() se encarga de inicializar el tablero con el tamaño definido con todas las posiciones a 0
    Este código modifcia el código inicial donde se han creado diferentes funciones para organizar el codigo de mejor manera.
